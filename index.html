2.1 Layout Systems: Flexbox

Flexbox Basics

Flexbox is a one-dimensional layout method for arranging items in rows or columns.

/* Flexbox Container */ .container { display: flex; flex-direction: row; /* or column */ justify-content: space-between; align-items: center; gap: 20px; } /* Flex Items */ .flex-item { flex: 1; /* grow to fill available space */ }

Common Flex Properties

  • justify-content: Alignment along main axis (start, center, end, space-between, space-around)
  • align-items: Alignment along cross axis (start, center, end, stretch)
  • flex-wrap: Allow items to wrap to new lines
  • gap: Space between flex items

2.2 Layout Systems: CSS Grid

Grid Basics

CSS Grid is a two-dimensional layout system for creating complex layouts.

.container { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-gap: 20px; }

Grid Template Areas

.container { display: grid; grid-template-areas: "header header header" "sidebar main main" "footer footer footer"; }

When to Use Grid vs Flexbox

  • Use Flexbox: For one-dimensional layouts (rows OR columns)
  • Use Grid: For two-dimensional layouts (rows AND columns)
  • Use Flexbox: For component-level layouts
  • Use Grid: For page-level layouts

2.3 Positioning Systems

Position Values

  • static: Default positioning (normal document flow)
  • relative: Positioned relative to its normal position
  • absolute: Positioned relative to nearest positioned parent (other than static)
  • fixed: Positioned relative to viewport
  • sticky: Toggles between relative and fixed

Box Model

/* Box Model: Content + Padding + Border + Margin */ .box { width: 200px; /* Content width */ padding: 20px; /* Inner spacing */ border: 2px solid; /* Border */ margin: 10px; /* Outer spacing */ }

Consistent Spacing Tokens

Use consistent spacing values:

  • Extra Small: 4px
  • Small: 8px
  • Medium: 16px
  • Large: 24px
  • Extra Large: 32px
  • XXL: 48px

CSS Custom Properties for Spacing

:root { --spacing-xs: 4px; --spacing-sm: 8px; --spacing-md: 16px; --spacing-lg: 24px; --spacing-xl: 32px; } .card { padding: var(--spacing-md); margin-bottom: var(--spacing-lg); }

4. Responsiveness

Mobile-First Approach

/* Mobile styles (default) */ .card { padding: 16px; font-size: 14px; } /* Tablet and up */ @media (min-width: 768px) { .card { padding: 24px; font-size: 16px; } } /* Desktop and up */ @media (min-width: 1024px) { .card { padding: 32px; font-size: 18px; } }

Responsive Units

  • %: Relative to parent element
  • vw/vh: Viewport width/height
  • rem: Relative to root font size
  • em: Relative to parent font size
  • px: Fixed pixels (use sparingly)

4.1 Responsive Patterns

Flexible Grids

.grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: 20px; }

Min/Max Constraints

.card { width: 100%; min-width: 200px; max-width: 400px; } .text-content { width: clamp(300px, 50%, 800px); /* min: 300px, preferred: 50%, max: 800px */ }

Responsive Typography

/* Fluid Typography */ h1 { font-size: clamp(1.5rem, 5vw, 3rem); } /* Responsive Type Scale */ :root { --fs-300: clamp(0.875rem, 2vw, 1rem); --fs-400: clamp(1rem, 2.5vw, 1.25rem); --fs-500: clamp(1.25rem, 3vw, 1.75rem); --fs-600: clamp(1.5rem, 4vw, 2.5rem); }

Common Responsive Patterns

  • Stack on Mobile: Horizontal layouts become vertical
  • Hide/Show: Different content for different screens
  • Reorder: Change visual order based on screen size
  • Adapt: Change component style (e.g., hamburger menu)

5. Identifying Reusable UI

Component Thinking

Look for patterns that repeat across your interface:

  • Visual Patterns: Similar styles, layouts, or behaviors
  • Functional Patterns: Similar interactions or purposes
  • Content Patterns: Similar data structures or displays

Best Practices Summary

Layout

  • Use Flexbox for one-dimensional layouts
  • Use Grid for two-dimensional layouts
  • Avoid absolute positioning unless necessary
  • Use semantic HTML for better structure

Spacing

  • Use consistent spacing tokens throughout
  • Prefer padding for internal spacing, margin for external
  • Use CSS custom properties for maintainable spacing
  • Consider the 8-point grid system

Responsiveness

  • Design mobile-first
  • Use flexible units (%, rem, vw/vh)
  • Test on real devices, not just browser tools
  • Consider touch targets for mobile (min 44x44px)

Reusability

  • Think in components, not pages
  • Create a consistent design system
  • Document your components
  • Use BEM or similar naming conventions
